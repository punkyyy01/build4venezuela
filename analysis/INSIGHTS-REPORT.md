### acopiovzla
Acopio VZLA (also branded "Saman") is a React/TypeScript SPA backed by Supabase that maps humanitarian aid collection centers (centros de acopio) in Venezuela post-earthquake. Users can browse centers publicly, see what supplies are needed, follow real-time updates per center, leave comments, and post shelter listings (anuncios de hospedaje). The project was built in roughly 2 days by a single developer, is deployed on Vercel, and shows unusually strong engineering discipline for a hackathon: 10 versioned DB migrations, 98.99% line test coverage via Vitest + MSW, well-organized feature-based folder structure, and proper Supabase RLS on all tables. The main gap is no CI/CD and no server-side rate limiting or logging.

- **Type:** Frontend SPA (React + Vite) with Supabase BaaS
- **Stack:** FE: React 18, TypeScript, Vite, Tailwind CSS, TanStack Query v5, React Router v6, lucide-react | BE: Supabase Auth, Supabase Realtime, Supabase Storage, PostgreSQL (via Supabase) | DB: PostgreSQL, PostGIS (initial migration; coords removed in migration 00002) | AI: 
- **Architecture:** Feature-based SPA with BaaS (Supabase). No custom server — all business logic lives in React hooks and Supabase RLS/DB triggers. · ORM: false (Supabase JS client (supabase-js v2) with direct table queries and RPC calls. No ORM. Typed via generated database.generated.ts + hand-written db.ts types.) · API: No custom REST/GraphQL API. The frontend calls Supabase REST endpoints directly. PostgreSQL triggers handle server-side side effects (e.g., notification on comment insert via notificar_comentario() trigger). 10 versioned SQL migrations managed via Supabase CLI.
- **Scores:** maturity 3/5 · prod-ready 3/5 · org 4/5 · viability 4/5 · product 3/5 · diffusion 3/5

The SQL migration system makes schema changes traceable.

### aid-trace
AidTrace / RastroAyuda is a legitimately built, offline-first humanitarian aid tracking PWA that writes immutable proof-of-custody records to a deployed smart contract on Celo Mainnet. The contract is live and verified on Celoscan. The app stores batches locally (localStorage + IndexedDB via Service Worker) and syncs them to the blockchain when connectivity returns. A Telegram bot integration via the Zavu SDK allows field operators to record events via text message. The backend relay is two Vercel serverless functions (api/zavu.mjs, api/timeline.mjs) using viem for chain interaction. A distributed write-locking mechanism (Supabase Postgres or Upstash Redis) serializes concurrent Celo writes. This is real, working code — not boilerplate — deployed at aidtrace-rastroayuda.vercel.app.

- **Type:** Offline-first PWA + blockchain audit trail for humanitarian aid supply chain tracking
- **Stack:** FE: Vanilla JS, HTML5, CSS3, Service Worker, IndexedDB, QR code (qrcode.js), PWA / manifest.webmanifest | BE: Node.js ESM serverless (Vercel), Express (listed in package.json), Zavu SDK (Telegram/WhatsApp/SMS relay) | DB: Celo Mainnet smart contract (AidTraceLedger.sol), localStorage (client-side state), IndexedDB (service worker sync queue), Supabase Postgres (optional distributed lock), Upstash Redis (optional distributed lock) | AI: 
- **Architecture:** Static PWA with serverless API relay and on-chain event log · ORM: false (No ORM. Blockchain state is read via viem getLogs() chunked by block range. Client state is raw JSON in localStorage. Service worker uses raw IndexedDB API.) · API: Two Vercel serverless functions: POST /api/zavu (browser relay + Zavu Telegram webhook) and GET /api/timeline (reads AidTraceEvent logs from Celo). Schema discriminated by event.schema field on the relay endpoint.
- **Scores:** maturity 3/5 · prod-ready 3/5 · org 4/5 · viability 4/5 · product 4/5 · diffusion 4/5
- **Signals:** 0★ · 49 commits · 3238 LOC · 1 contributors · no-license

AidTrace directly targets both: field operators scan a QR label to record each handoff even with zero connectivity, and when internet returns the record is permanently written to a public blockchain anyone can audit. It is not a landing page — it is the actual product. The smart contract AidTraceLedger is deployed and verified on Celo Mainnet (0xaf5c40e82ac9255479a1f447e81992b71c4f4934). The Telegram bot command interface (CELO1 entregar...) is documented and wired via the Zavu SDK relay.
